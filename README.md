# Paws & Whiskers 🐾
### CC14 – Product Landing Page Activity

## Project Overview
Paws & Whiskers is a fictional e-commerce landing page created for the **CC14 Product Landing Page Activity**. The purpose of this project is to demonstrate the ability to design and develop a responsive landing page using HTML, CSS, and JavaScript.

The website represents a fictional online pet shop that offers products for cats and dogs. The landing page highlights featured products, promotions, and encourages users to explore the store through call-to-action buttons.

## Assignment Objective
By completing this project, the following objectives were achieved:

- Understand the structure and components of a landing page
- Implement HTML and CSS to create a visually appealing e-commerce page
- Apply basic responsive design principles for different screen sizes


## Website Features

### Header
The header contains:
- Store logo (Paws & Whiskers)
- Navigation menu:
  - Home
  - Products
  - Gallery
  - Blog
  - About Us
  - Contact Us

### Hero Section
The hero section includes:
- A large banner area
- Headline and subheadline introducing the store
- A **call-to-action button** such as **Shop Now**

### Featured Products Section
This section showcases several pet products with the following details:
- Product name
- Product image
- Product price
- "Add to Cart" button

### Testimonials Section
This section contains customer reviews and ratings to simulate real user feedback.

### Footer
The footer contains:

Contact Information:
- Email
- Phone number

Additional links:
- Terms & Conditions
- Privacy Policy

Social media icons are also included.

## Note on JavaScript Usage

The original requirements for this activity only required the use of **HTML and CSS** to build the product landing page. However, **JavaScript was also used in this project for additional creativity and learning purposes**.

Although JavaScript was **not a required part of the activity**, it was included to enhance the overall functionality and improve the user experience of the website. This was done as an opportunity to further develop my knowledge and skills in front-end web development.

The use of JavaScript in this project is therefore **optional and supplementary**, and it does not replace the primary objective of demonstrating the use of **HTML and CSS for building a responsive landing page**.
